# Flow — Interaction Vocabulary

The naming system for Flow should feel **musical, fluid, and alive** — not technical or mechanical.

## Core Feature Names

| Name | Category | Description |
|------|----------|-------------|
| **Seed** | Input | The starting material — your raw idea, image, or sketch |
| **Pulse** | State | The live energy of the session; what's active and resonating |
| **Branch** | Action | Expand an idea into new directions |
| **Remix** | Action | Generate divergent versions of a concept |
| **Refine** | Action | Narrow and sharpen an idea |
| **Anchor** | Action | Lock in a node as a fixed point of reference |
| **Flow Bar** | UI Element | Session timeline showing ideation rhythm over time |
| **Canvas** | UI Element | The central generative workspace |
| **Outputs** | UI Element | Right panel — structured results |
| **Flow Mode** | State | Deep work / immersive mode with minimal chrome |

## Interaction Language (Copy Principles)

- Use **verbs that feel like movement**: branch, pulse, flow, anchor, drift, crystallize
- Avoid clinical language: no "generate," "submit," "process"
- Prefer poetic shorthand: "Let it branch" over "Create variations"
- Status messages should feel like the tool is *alive*: "Listening...", "Thinking in color...", "A new branch is forming..."

## Potential Additions

- **Drift** — loosely explore without anchoring anything
- **Crystallize** — collapse a cloud of ideas into a single clear concept
- **Echo** — surface a related idea from earlier in the session
- **Hum** — low-key background AI suggestion (ambient, non-intrusive)
- **Surge** — high-energy rapid-fire ideation mode

## Voice & Tone

> A thoughtful creative collaborator who's very present, never rushed, occasionally poetic, and always in service of *your* idea — not its own.
